# Icon library

Each subfolder is one icon group (the folder name becomes the group shown in the icon picker);
every `*.svg` is embedded into `Dependinator.UI.dll` and loaded by `IconLibrary.cs`. The file
name is the icon's id: it must equal the root `<svg id="...">` and be unique across all groups.

## Default, General

Hand-authored 24x24 icons in the shared violet palette; these get tint color variants generated
at load (see `IconLibrary`).

## Azure, Aws, Google

Curated subsets of the official free cloud-provider icon sets, generated by `./import-icons`
from the manifests in `scripts/cloud-icons/*.manifest` — don't edit these files by hand. The
svgs are normalized on import (root id set, internal ids icon-prefixed, Google's css classes
inlined, `<title>`/width/height stripped); the official colors are kept, so no tint variants
are generated for them.
